# Hut list and coordinate generation

This document explains how the hut list and latitude/longitude coordinates for the Alpine huts map were generated.

The process had two separate steps:

```text id="rh7xsn"
1. Extract the list of huts from the reservation system
2. Geocode those huts to obtain latitude and longitude
```

The coordinates used in the project were **not taken directly from hut-reservation.org**. The reservation system was used first to obtain the list of available huts. The hut names were then geocoded separately using Google Sheets / Google Apps Script.

## Step 1: Extracting the original hut list

The original hut list came from the reservation-system scraper.

Inside the scraper, there was an initial piece of code that extracted the list of huts from the reservation system. This code was later commented out because the hut list only needed to be generated once, while the main scraper was used repeatedly for availability scraping.

The extracted hut list produced the original column:

```text id="d3birw"
huts
```

Example values:

```text id="xt4kic"
Adamek-Hütte, AT
Aarbiwak SAC, CH
Aclas Dafora, CH
```

This `huts` column contained the full reservation-system hut label, including both the hut name and the country suffix.

After extracting the `huts` column, it was split into two cleaner columns:

```text id="t4cx2s"
hut_name
country
```

For example:

```text id="dcvuto"
huts                 hut_name        country
Adamek-Hütte, AT     Adamek-Hütte    AT
Aarbiwak SAC, CH     Aarbiwak SAC    CH
Aclas Dafora, CH     Aclas Dafora    CH
```

So the three original identifying columns were:

```text id="cmbefa"
huts
hut_name
country
```

These columns became the base hut table used for all later steps.

## Step 2: Geocoding hut coordinates

The latitude and longitude coordinates were generated in a separate geocoding step.

The input was the hut table with these columns:

```text id="fm1sxq"
huts
hut_name
country
```

Additional empty columns were added for the geocoding output:

```text id="byuibx"
lat
lon
status
formatted_address
query_used
```

The final Google Sheet therefore had the following structure:

```text id="mo3uo8"
huts
hut_name
country
lat
lon
status
formatted_address
query_used
```

The geocoding script used `hut_name` and `country` to create several Google geocoding queries. If one query returned a valid result, the script wrote latitude, longitude, the formatted address, and the successful query back into the sheet.

The script skipped rows where `lat` was already filled, so it could be safely run multiple times.

## Full workflow

The complete workflow was:

```text id="c4bx3i"
reservation-system scraper
        ↓
extract full hut labels
        ↓
huts column
        ↓
split into hut_name and country
        ↓
Google Sheets geocoding
        ↓
lat/lon/status/formatted_address/query_used
        ↓
export as CSV
        ↓
convert to GeoJSON
        ↓
use in frontend map
```

## Important distinction

The reservation system was the source for the **hut list**, not for the final map coordinates.

The coordinates were generated by Google geocoding from the hut names. This is visible from the geocoding metadata columns:

```text id="cuzlwy"
status
formatted_address
query_used
```

These columns show which Google geocoding query was used and what address Google returned.

## Google Apps Script used for geocoding

```javascript id="emb70r"
function geocodeHuts() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("huts");
  const data = sheet.getDataRange().getValues();
  const header = data[0];

  const col = {};
  header.forEach((h, i) => col[h] = i);

  const countryNames = {
    "AT": "Austria",
    "CH": "Switzerland",
    "IT": "Italy",
    "DE": "Germany",
    "SI": "Slovenia",
    "FR": "France",
    "LI": "Liechtenstein"
  };

  const geocoder = Maps.newGeocoder()
    .setLanguage("de")
    .setBounds(44.0, 5.0, 49.8, 18.5); // roughly the Alps

  const MAX_PER_RUN = 100;
  let done = 0;

  for (let i = 1; i < data.length; i++) {
    const hutFull = data[i][col["huts"]];
    const hutName = data[i][col["hut_name"]];
    const countryCode = data[i][col["country"]];
    const existingLat = data[i][col["lat"]];

    if (!hutName || existingLat) continue;

    const country = countryNames[countryCode] || countryCode;

    const queries = [
      `${hutName} ${country}`,
      `${hutName} ${country} Hütte`,
      `${hutName} ${country} Alpenverein`,
      `${hutName} ${country} mountain hut`,
      hutFull
    ];

    let found = false;

    for (const q of queries) {
      try {
        const response = geocoder.geocode(q);

        if (response.status === "OK" && response.results.length > 0) {
          const result = response.results[0];
          const loc = result.geometry.location;

          sheet.getRange(i + 1, col["lat"] + 1).setValue(loc.lat);
          sheet.getRange(i + 1, col["lon"] + 1).setValue(loc.lng);
          sheet.getRange(i + 1, col["status"] + 1).setValue("OK");
          sheet.getRange(i + 1, col["formatted_address"] + 1).setValue(result.formatted_address);
          sheet.getRange(i + 1, col["query_used"] + 1).setValue(q);

          found = true;
          break;
        }
      } catch (e) {
        sheet.getRange(i + 1, col["status"] + 1).setValue("ERROR: " + e.message);
      }

      Utilities.sleep(250);
    }

    if (!found) {
      sheet.getRange(i + 1, col["status"] + 1).setValue("NOT_FOUND");
    }

    done++;
    Utilities.sleep(300);

    if (done >= MAX_PER_RUN) break;
  }
}
```

## Re-running the geocoding

To run the geocoding again:

1. Open the Google Sheet.
2. Make sure the sheet is named `huts`.
3. Make sure the columns are present:
   `huts`, `hut_name`, `country`, `lat`, `lon`, `status`, `formatted_address`, `query_used`.
4. Clear `lat` and `lon` for any huts that should be geocoded again.
5. Run `geocodeHuts()` from Apps Script.
6. Repeat until all huts have either `OK` coordinates or a clear failure status.

## Quality checks

After geocoding, manually check the results before publishing.

Recommended checks:

```text id="t7nggr"
status != OK
formatted_address in wrong country
duplicate coordinates
coordinates outside the Alps
suspicious query_used values
```

The most useful columns for checking are:

```text id="w5cd7x"
status
formatted_address
query_used
```

A row can have `status = OK` and still be wrong if Google matched the hut name to the wrong place.

## GeoJSON conversion

After checking the coordinates, the table was exported and converted into GeoJSON for the frontend map.

Important: GeoJSON uses coordinate order:

```text id="sxaywm"
[longitude, latitude]
```

So:

```text id="tr4tqs"
lat = 47.4922493
lon = 13.5838135
```

must become:

```json id="7s5kft"
"coordinates": [13.5838135, 47.4922493]
```

This GeoJSON file is what the frontend map reads when displaying hut markers.
